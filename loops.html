Zanke


Zanke so priročne, če želite vedno znova zagnati isto kodo, vsakič z drugo vrednostjo. Pogosto se pri delu z nizi zgodi naslednje:

Namesto pisanja:
besedilo + = avtomobili [0] + "...";
besedilo + = avtomobili [1] + "...";
besedilo + = avtomobili [2] + "...";
besedilo + = avtomobili [3] + "...";
besedilo + = avtomobili [4] + "...";
besedilo + = avtomobili [5] + "...";
Lahko napišete:
var i;
za (i = 0; i besedilo + = avtomobili [i] + "...";
}

Različne vrste zank
JavaScript podpira različne vrste zank:
for - večkrat ponovi blok kode
for / in - zanke skozi lastnosti predmeta
for / of - zanke skozi vrednosti iterabilnega predmeta
while - se vrti skozi blok kode, medtem ko je določen pogoj resničen
do / while - tudi zanke skozi blok kode, medtem ko je določen pogoj resničen

Za zanka
Zanka for ima naslednjo sintakso:

za (stavek 1; stavek 2; stavek 3) {
// blok kode, ki se izvede
}

Stavek 1 se izvede (enkrat) pred izvajanjem bloka kode.
Stavek 2 opredeljuje pogoj za izvajanje kodnega bloka.
Stavek 3 se izvede (vsakič) po izvedbi bloka kode.

Primer:
za (i = 0; i <5; i ++) {
text + = "Število je" + i + "...";
}

Iz zgornjega primera lahko preberete:
Izjava 1 nastavi spremenljivko pred zagonom zanke (var i = 0).
Izjava 2 opredeljuje pogoj za zagon zanke (mora biti manjša od 5).
Izjava 3 poveča vrednost (i ++) vsakič, ko se izvede blok kode v zanki.

Stavek 1
Običajno boste uporabili stavek 1 za inicializacijo spremenljivke, uporabljene v zanki (i = 0). Temu ni vedno tako, JavaScriptu je vseeno. Izjava 1 ni obvezna.
V stavku 1 (ločeno z vejico) lahko sprožite veliko vrednosti:

Primer:
za (i = 0, len = cars.length, text = ""; i besedilo + = avtomobili [i] + "...";
}

Izjavo 1 lahko izpustite (na primer, ko so vaše vrednosti nastavljene pred zagonom zanke):

Primer:
var i = 2;
var len = cars.length;
var text = "";
za (; i besedilo + = avtomobili [i] + "...";
}

Stavek 2
Stavek 2 se pogosto uporablja za oceno stanja začetne spremenljivke. Temu ni vedno tako, JavaScriptu je vseeno. Tudi stavek 2 je neobvezen. Če stavek 2 vrne true, se zanka začne znova, če vrne false, se zanka konča.

Če izpustite izjavo 2, morate zagotoviti zanko znotraj zanke. V nasprotnem primeru se zanka ne bo nikoli končala. To bo zrušilo vaš brskalnik.

Vaja 1/5.
Ustvari zanko, ki teče od 0 do 9.

Vaja 2/5.
Ustvari zanko, ki bo izpisala vsa soda števila od 0 do 25.

Vaja 3/5.
Ustvari zanko, ki bo seštela vsa števila od 0 do 10. Seštevek naj se izpiše.

Vaja 4/5.
Ustvari program z zanko while, ki izpiše vsa števila od 0 do 9.

Vaja 5/5.
Ustvari zanko, ki bo izpisala vsa liha števila od 0 do 25.





Izjava 3
Stavek 3 pogosto poveča vrednost začetne spremenljivke. Temu ni vedno tako, JavaScriptu je vseeno in stavek 3 ni obvezen. Izjava 3 lahko naredi karkoli, na primer negativni prirastek (i--), pozitivni prirastek (i = i + 15) ali kar koli drugega. Izjava 3 je lahko tudi izpuščena (na primer pri povečevanju vrednosti znotraj zanke):

Primer:
var i = 0;
var len = cars.length;
za (; i len; ) {
besedilo += avtomobili[i] + "...";
i ++;
}

Medtem ko zanka
Zanka while kroži skozi blok kode, dokler je določen pogoj resničen.
Sintaksa:
medtem ko (stanje) {
// blok kode, ki se izvede
}
V naslednjem primeru se bo koda v zanki izvajala znova in znova, dokler bo spremenljivka (i) manjša od 10:

Primer:
medtem ko (i <10) {
text + = "Število je" + i;
i ++; }

Zanka Do / While
Zanka do / while je različica zanke while. Ta zanka bo enkrat izvedla blok kode, preden bo preverila, ali je pogoj resničen, nato bo zanko ponavljala, dokler je pogoj resničen.
Sintaksa:
naredi {
// blok kode, ki se izvede
}
medtem ko (stanje);

Spodnji primer uporablja zanko do / while. Zanka se bo vedno izvedla vsaj enkrat, tudi če je pogoj neresničen, ker se blok kode izvede pred preizkusom pogoja:

Primer:
naredi {
text + = "Število je" + i;
i ++;
}
medtem ko (i <10);

Ne pozabite povečati spremenljivke, ki je uporabljena v pogoju, sicer se zanka ne bo nikoli končala!
